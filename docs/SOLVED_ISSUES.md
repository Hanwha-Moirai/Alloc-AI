# 해결된 문제점 정리

## 1. 데이터 경계 불분명 문제
- 해결
- 데이터 출처 명확화
  - 회의록 / 일정 변경 로그 / 간트 / 이전 주간 보고: MariaDB
  - 사내 일정 리스크 판단 기준 문서: Qdrant(Vector DB)
- RAG 입력 데이터의 출처 혼선 제거

---

## 2. 근거 추적성 방향성
- 해결
- 일정 리스크 후보 작성 시
  - 관련 태스크 명시
  - 근거 규정 문서 발췌
  - 규정 문서 내 섹션/출처 명시
- “왜 이 리스크가 도출되었는지”에 대한 최소 설명 가능

---

## 3. 멱등성 처리 위치
- 해결
- Spring Framework가 주간 보고 생성 API의 단일 진입점
- 멱등성 처리 책임을 Spring으로 일원화
- FASTAPI는 순수 생성 역할만 수행

---

## 4. 시스템 책임 분리
- 해결
- Spring: 인증, 권한, 멱등성, 저장, 단일 진실 소스
- FASTAPI: 조회 + RAG 기반 생성
- 프론트엔드가 생성 결과를 직접 받지 않음
